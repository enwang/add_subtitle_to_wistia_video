## Visual QA Rule

For any UI, layout, typography, PDF styling, or other visual presentation change:

- Generate the actual output first.
- Review the rendered result directly before replying.
- Do not reply to the user until the visual result has been checked and is reasonably polished.
- If the result is clearly off, iterate and fix it before responding.

This applies especially to:

- PDF summaries
- Page layout and spacing
- Fonts and typography
- Visual hierarchy
- Screens, previews, and other user-facing rendered output
